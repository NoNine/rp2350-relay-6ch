# RP2350-Relay-6CH Hardware Info

Hardware reference for the Waveshare RP2350-Relay-6CH board variant using an
RP2350B MCU. The `RP2350-Relay-6CH-W` assembly also includes a Raspberry Pi
Radio Module 2 (RM2).

Sources:

- Product brief: <https://www.waveshare.net/wiki/RP2350-Relay-6CH>
- Schematic: <https://www.waveshare.net/w/upload/0/04/RP2350-Relay-6CH.pdf>
- Schematic metadata: 1-page Altium PDF, creation date 2025-06-23

## Main Components

| Function | Part / Notes |
| --- | --- |
| MCU | Raspberry Pi RP2350B |
| Wireless | Optional Raspberry Pi Radio Module 2 footprint/module on `RP2350-Relay-6CH-W` |
| Flash | W25Q128JVSIQ 128-Mbit QSPI flash |
| USB | USB Type-C device connector wired to RP2350B USB D+/D- |
| Relays | 6x HLS8L-DC5V relay channels |
| Relay isolation | PC817 optocouplers between MCU-side GPIO logic and relay-side drive |
| RS485 | SP485EEN transceiver, isolated through a Pi141E61 digital isolator |
| Buzzer | 3.3 V buzzer driven by NPN transistor |
| RGB LED | WS2812B-2020 on-board RGB LED |
| Power input | External screw terminal input, documented by Waveshare as DC 7 V to 36 V |
| Logic regulator | SGM2212-3.3 3.3 V LDO for MCU-side logic |
| Relay supply | Relay-5V rail generated from the external input through isolated DC/DC |

## Power Domains

| Net | Purpose |
| --- | --- |
| 5V / VSYS | USB/external board supply domain |
| 3V3 | RP2350B, flash, optional RM2, logic, buzzer, RGB LED |

## GPIO Assignment

| Signal | RP2350B GPIO | Direction | Notes |
| --- | ---: | --- | --- |
| `TXD0` | GPIO0 | Output | Routed to Pico-compatible header |
| `RXD0` | GPIO1 | Input | Routed to Pico-compatible header |
| `CH1` | GPIO26 | Output | Relay channel 1 control, active high |
| `CH2` | GPIO27 | Output | Relay channel 2 control, active high |
| `CH3` | GPIO28 | Output | Relay channel 3 control, active high |
| `CH4` | GPIO29 | Output | Relay channel 4 control, active high |
| `CH5` | GPIO30 | Output | Relay channel 5 control, active high |
| `CH6` | GPIO31 | Output | Relay channel 6 control, active high |
| `BUZZER` | GPIO23 | Output | Drives transistor T1; active high at GPIO turns buzzer on |
| `TXD1` | GPIO24 | Output | UART1 TX into isolated RS485 path |
| `RXD1` | GPIO25 | Input | UART1 RX from isolated RS485 path |
| `TXD1EN` | Derived logic | Output | RS485 driver-enable net generated by the 74HC04D helper circuit from the UART TX path |
| `RGB_CTRL` | GPIO36 | Output | WS2812B data input through 470-ohm resistor |
| `USB_BOOT` | QSPI_SS | Input | Boot button pulls QSPI_SS low through 1 kOhm |
| `RUN` | RUN | Input | Reset/run button pulls RUN low |
| `SWCLK` | SWCLK | Debug | SWD clock test/button net |
| `SWD` | SWD | Debug | SWD data |

Relay notes:

- Each relay GPIO feeds the relay side through a 0-ohm link, optocoupler, SS8050 transistor driver, flyback diode, and red indicator LED.
- The schematic labels relay outputs `CH1` through `CH6`; the GPIO side is 3.3 V logic and the relay coil side is on `Relay-5V`.
- Treat relay GPIOs as active high unless board testing proves otherwise.

## RS485

| RS485 Signal | RP2350B Side | Isolation / Transceiver Side |
| --- | --- | --- |
| TX data | `TXD1` / GPIO24 | Pi141E61 `TXD'` to SP485EEN DI |
| RX data | `RXD1` / GPIO25 | SP485EEN RO through Pi141E61 `RXD'` |
| Direction enable | Hardware-generated `TXD1EN` | Pi141E61 `TXDEN'` to SP485EEN DE and `/RE` control |
| Bus A | `RS485A` | Connector J10 pin 1 |
| Bus B | `RS485B` | Connector J10 pin 2 |
| Shield/protection | `Earth` | Connector J10 pin 3 |

Protection and termination:

- `RS485A` / `RS485B` include resettable fuses, TVS/ESD devices, and common-mode/surge protection.
- The schematic includes a 120-ohm resistor between the isolated bus-side nets.
- RS485 logic is on isolated `SGND`; do not short `GND` and `SGND` in firmware assumptions.

## Optional Raspberry Pi Radio Module 2

The schematic names this section `RM2` / `RPI RM2`. It applies to the optional
`RP2350-Relay-6CH-W` wireless assembly.

| RM2 Function | RP2350B GPIO / Net | RM2 Ball |
| --- | --- | --- |
| `WL_ON` | GPIO32 | B5 `WLON` |
| `WL_D` | GPIO33 | A5 `D1` |
| `WL_CS` | GPIO34 | B2 `CS` |
| `WL_CLK` | GPIO35 | A3 `SCLK` |
| `WL_D` through R66 470 ohm | GPIO33-derived data path | A6 `DO` |
| `VBUS_SENSE` | VBUS through R68/R69 divider | C3 `GPIO2` |
| `VDDBAT` / `VDDIO` | 3.3 V through ferrite bead | C2/B7 |

RM2 notes:

- Several RM2 nets are connected through 10 kOhm or 470-ohm resistors; check the schematic before changing default pulls or drive strength.
- RM2 module operation and Wi-Fi support have not been verified and are deferred.

## Zephyr Board-Port Notes

- Build relay firmware with `waveshare_rp2350_relay_6ch/rp2350b/m33` by
  default. Use `waveshare_rp2350_relay_6ch/rp2350b/m33/w` only for the
  optional RM2 Wi-Fi assembly.
- Use Zephyr's Raspberry Pi Pico 2 W board definition at
  `${ZEPHYR_WORKSPACE:-$HOME/zephyrproject}/zephyr/boards/raspberrypi/rpi_pico2`
  as the base reference for common RP2350 board configuration. The target board
  differs by using an RP2350B MCU and the Waveshare pin assignment below.
- Do not use the Raspberry Pi Pico 2 W target for this hardware. Its Wi-Fi
  devicetree assigns GPIO23, GPIO24, GPIO25, and GPIO29 to Pico 2 W-specific
  functions that conflict with this board's buzzer, RS485 path, and `CH4`.
- Model relays as GPIO outputs with an active-high polarity until measured on hardware.
- Model the buzzer as an active-high GPIO output on GPIO23.
- Model the WS2812 as a one-wire LED strip data signal on GPIO36.
- Model RS485 on UART1: `tx-pin = GPIO24`, `rx-pin = GPIO25`. The schematic shows the driver-enable timing handled in hardware through `TXD1EN`, so a Zephyr DE GPIO may not be needed.
- Be careful assigning GPIO26, GPIO27, GPIO28, GPIO29, GPIO30, and GPIO31 to other functions because they are relay outputs.
- For wireless support, use the `/w` board variant and verify the board
  assembly option before enabling RM2 driver nodes.
